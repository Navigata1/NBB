## Patterns Identified

### Pattern 1: Verification as a dedicated phase
Running URL checks, plugin verification, and cross-reference audits as a distinct phase (rather than inline during writing) caught errors that went unnoticed during the creation passes. This pattern should be standard for all batches.

### Pattern 2: Version-string matching in hooks is fragile
The pre-write hook matches `*v6*` and `*v2*` in paths. When versions bump, the hook breaks silently. Directory-based matching (`north-star-blueprint/` and `master-build-framework/`) would be more durable.

### Pattern 3: Struck-through URLs preserve audit trail
SKILLS_REGISTRY marks broken URLs with strikethrough and dates rather than deleting them. This preserves the audit trail and is a good convention for governance documents.

### Pattern 4: Sub-agent parallelism for context gathering
Using 2 sub-agents in parallel (one for git context, one for file review) cut retro prep time roughly in half. This pattern works well for any information-gathering phase.

### Learning B — claude.md Rule

```

REASON: Version strings change across releases; directory names are stable. The pre-write hook matching `*v6*` will need updating when Blueprint reaches v7.
EXAMPLE: Match `north-star-blueprint/*.md` instead of `*NORTH_STAR_BLUEPRINT_v6*`.
```
